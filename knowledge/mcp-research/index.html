📅 发展时间线

2024 年 11 月
MCP 开源发布 — Anthropic 发布 Model Context Protocol，定位为 AI 应用连接数据源和工具的开放标准
2025 年 3 月
Streamable HTTP — 引入可流式 HTTP 传输，替代 SSE，支持 AWS Lambda 等无服务器部署，实现远程 MCP
2025 年 6 月
重大规范更新 — 发布 Elicitation（服务器请求用户输入）、Tool Output Schemas（工具输出类型定义）、OAuth 2.1 授权规范
2025 年 8 月
Visual Studio GA — MCP 支持在 Visual Studio 中正式 GA，Microsoft 全面拥抱 MCP
2025 年 12 月
Linux Foundation AAIF 成立 — Anthropic 将 MCP 捐赠给 Linux 基金会 Agentic AI Foundation，确立行业标准地位
2026 年 4 月（计划）
MCP Dev Summit NYC — 第一届大规模 MCP 开发者峰会，4 月 2-3 日纽约举办

核心原语

原语说明
🔧 Tools可执行函数（API 调用、数据库查询）
📦 Resources数据源（文件、数据库记录）
💬 Prompts交互模板（系统提示、工作流）

传输层

方式场景
Stdio本地进程通信（标准输入输出）
Streamable HTTP远程通信（支持云部署）

协议基于 JSON-RPC 2.0，支持能力协商和动态发现

⚔️ 协议竞争格局

协议发起方定位势头
MCP Anthropic → Linux Foundation AI ↔ 工具/数据 连接标准 最强
A2A Google Agent ↔ Agent 通信 增长中
ACP Cisco / IBM Agent 协作框架 早期
AGENTS.md OpenAI → Linux Foundation Agent 行为规范 广泛采用

关键洞察：MCP 与 A2A 是互补关系而非竞争 —— MCP 负责"AI 连接工具"，A2A 负责"Agent 间通信"。两者可以共存。

  • 中 Sampling 攻击
  • 中 恶意安装工具

🔮 趋势判断

  1. 行业标准确立 — MCP 已从"协议提案"进化为"行业基础设施"，Linux Foundation 治理确保中立性
  2. 从本地走向远程 — Streamable HTTP 的引入使 MCP 可以连接云端服务，不再局限于本地进程
  3. 安全治理成为焦点 — OAuth 2.1、Registry 审核、工具签名将成为下一阶段重点建设方向
  4. 生态爆发临界点 — 10,000+ Server + 主流平台全面支持，类比 2015 年的 npm 生态
  5. AI 原生 Web 基石 — MCP 可能成为 AI 时代的 HTTP —— AI 应用通过 MCP 连接一切
